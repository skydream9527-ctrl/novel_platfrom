# Analysis Agent Design

## 来源

- 飞书文档：第一篇已学习文档
- 获取时间：2026-04-21

## 当前理解

1. 分析型 Agent 的核心不是会回答问题，而是能把分析经验蒸馏成可复用的方法与步骤。
2. 一个靠谱的分析 Agent 需要同时具备对话理解、分析框架调用、质量校验和输出规范化能力。
3. Skill 的价值在于把高频、稳定、可复用的分析套路显式化，而不是让模型每次都从零发挥。

## 可复用原则

- 将分析框架库显式写入 Agent 的长期知识
- 将趋势归因、漏斗拆解、同期群分析等套路封装成标准能力
- 在输出前做样本量、口径一致性、可复现性检查
- 输出不只给结论，还要给分析路径和限制条件

## 对本 Agent 的启发

1. 学习型 Agent 也应把“提炼方法”本身沉淀为知识。
2. 以后对外部文档做学习时，应优先抽取：框架、流程、边界、校验机制。
3. 长期看应该形成一套“知识提炼 Skill 库”。
