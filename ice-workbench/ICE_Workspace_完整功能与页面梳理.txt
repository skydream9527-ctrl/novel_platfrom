【0】产品一句话定位
--------------------------------------------------------------------------
面向互联网产品团队的"数据工作流工作台"，以 Web 应用形态运行，
通过 Agent + 对话 + 工具调用驱动 AB 实验、经营分析、灰度监控、
数据分析、波动归因等数据任务，并把团队知识、技能、模板沉淀为可复用资产。


【1】整体架构
--------------------------------------------------------------------------
ice_data_workspace_v2/
├── frontend/        React 19 + TypeScript + Vite + Zustand 单页应用
├── server/          FastAPI + SQLAlchemy(asyncio) + SQLite
├── agents/          5 个本地 Agent（Python 进程）+ 共享运行时
│   ├── data_analysis/   数据分析 Agent (NL-SQL + 多专家辩论)
│   ├── general_agent/   通用 Agent (集成所有能力)
│   ├── know_agent/      知识库 Agent (飞书 + Mify RAG)
│   ├── learn_agent/     学习 Agent (网页抓取/对比/沉淀)
│   └── shared/          共享运行时(runtime/llm_client/memory/...)
├── skills/          25 个本地 Skill 目录（自动发现）
├── files/           4 篇公共参考文档（使用指南、AB 规范等）
├── product_plan/    PRD v2/v3 + 13 个设计稿 HTML
├── users_data/      SQLite 数据库 + 上传/导出/备份目录
└── (运行时)
    └── frontend/dist 构建产物 → 由 server 静态托管，实现单端口部署

技术栈核心：
  前端：React 19 / TypeScript / Vite / Zustand / Axios / react-markdown
        / ECharts / react-router-dom v7 / CSS Modules + CSS Variables
  后端：FastAPI / SQLAlchemy 2.0 异步 / aiosqlite / passlib(bcrypt)
        / python-jose(JWT) / openai(SSE) / WebSocket
  Agent：httpx SSE LLM 客户端 / 自研 ToolRegistry / Function Calling

【3】核心概念词典
--------------------------------------------------------------------------
任务 Task             一次数据工作流，每个任务拥有独立 Workspace
工作空间 Workspace    任务的执行环境，包含 Agent、Skills、Files
工作范式 Paradigm     5 种预置任务模板，每种绑定唯一 Agent + 技能集
Agent                 公共能力实体，与范式 1:1 绑定，可对话进化
Skill                 可执行工具，含 tool_schema，由 LLM function-calling 调用
经验卡片 Experience   从对话中提炼的规则，审批后注入 Agent System Prompt
Agent 记忆 Memory     对话沉淀的结构化记忆，多次验证后升级为经验卡片
公共区 Commons        团队共享的 Agents / Skills / Files / 模板 / 任务

gray_release    淡紫     灰度监控 Agent  灰度指标查询 / 版本对比 / 质量评估 / 放量建议
data_analysis   灰绿     数据分析 Agent  SQL查询 / 数据清洗 / 可视化 / 建模 / 报告生成
wave_analysis   暗红     波动归因 Agent  异动检测 / 多维下钻 / 归因定位 / 影响评估
(open)          —        通用 Agent      不绑定范式，开放任务专用

7.10 WebSocket
──────────────────────────────────────────────────────────────────────────
WS   /api/v1/ws/conversations/{cid}
事件协议:
  ← user_message           前端发送（含 referenced_file_ids / referenced_skill_ids）
  → user_message_ack       回执
  → agent_typing           开始思考
  → tool_call_start        工具调用开始 {tool_name, arguments_summary}
  → tool_call_done         工具完成 {result_summary, chart_config?}
  → agent_message          流式 chunk
  → agent_message_done     回答结束
  → file_created           Agent 产出文件
  → suggested_followups    推荐追问
约束:
  Tool Calling 循环最多 5 轮
  单 Tool 超时 30s
  对话历史 context 20 条

【13】Tool Calling 完整执行流程
==========================================================================

用户消息 (含 referenced_file_ids / referenced_skill_ids)
   ↓
组装 LLM 请求
  · System Prompt = Agent.config.system_prompt
                   + "## 经验规则" + 已批准经验卡片(最多 20)
                   + "## 可用工具" + Skill 描述
  · tools[] = agent_skills 绑定 + 用户引用 Skill 的 tool_schema
  · 文件内容注入: CSV ≤100 行全量；>100 行 schema+前 20 行；MD/TXT 全量
   ↓
httpx SSE 调用 LLM
   ↓
├── 文本 chunk → ws agent_message
└── tool_calls → 执行循环(≤5 轮):
      ↓ ws tool_call_start
      ToolExecutor:
        1. 查 tool_entry → 动态 import
        2. SQL 类先过安全审计(allow/warn/block)
        3. 执行函数(共享 ToolExecutionContext)
           shared_data[kyuubi_sql_query/auto_drilldown/...]
        4. 单 Tool 30s 超时
      ↓ ws tool_call_done (含 chart_config?)
      result 回传 LLM → 继续生成
      ├── 又 tool_calls → 继续(≤5 轮)
      └── 文本 → ws agent_message → done

ToolExecutionContext 跨 Skill 共享上下文，跨对话隔离。

【14】Agent 进化机制（经验卡片）
==========================================================================

用户对话 → Agent 回复
   ↓
异步分析对话质量:
  · 用户确认有效的分析路径
  · 高质量回答（👍 标记）
  · 反复被引用的结论
   ↓
提炼写入 AgentMemory
   ↓
记忆积累/多次验证 → 自动升级为 ExperienceCard(draft)
   ↓
管理员审批 → approved
   ↓
注入 Agent System Prompt 的"经验规则"章节(最多 20 条)
   ↓
Agent 后续对话行为进化
