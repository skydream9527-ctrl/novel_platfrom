# AI 文字创作平台 — 功能规划文档

> 对标 NotebookLM 设计理念，结合小说/剧本/分镜创作场景，梳理当前平台缺失功能并规划优先级。

---

## 一、对标分析：NotebookLM 核心能力

| 能力 | NotebookLM 实现 | 本平台现状 |
|------|----------------|-----------|
| 素材导入 | 支持 PDF、Google Docs、网页、YouTube、音频等 50+ 来源 | ❌ 无素材系统 |
| 源关联 AI | 所有回答基于上传素材，减少幻觉 | ❌ AI 仅靠对话上下文，无素材锚点 |
| 引用溯源 | 回答中内联引用，点击跳转原文 | ❌ 无引用机制 |
| 自动摘要 | 上传后自动生成摘要、关键主题 | ❌ 无自动摘要 |
| 学习指南 | 从素材自动生成学习指南 | ❌ 无 |
| FAQ 生成 | 自动从素材提取常见问答 | ❌ 无 |
| 时间线 | 自动从素材提取事件时间线 | ❌ 无 |
| 思维导图 | 可视化概念关联 | ❌ 无 |
| 音频概览 | 双主持人播客式音频摘要，可自定义 | ❌ 无 |
| 多笔记本 | 每个项目独立笔记本，组织多组素材 | ⚠️ 有任务(Task)概念，但无素材层 |
| 协作分享 | 笔记本可分享给团队成员 | ❌ 无协作功能 |
| 工作室面板 | 统一管理生成的各种输出 | ❌ 无统一输出管理 |

---

## 二、当前平台能力盘点

### 已有功能
- 用户注册/登录（JWT 双 Token）
- 角色权限（管理员/普通用户）
- 任务管理（创建/删除/列表，支持小说/剧本/分镜三种类型）
- 章节管理（创建/编辑/删除，自动版本号）
- Markdown 编辑器 + 实时预览
- AI 对话助手（WebSocket 流式，按任务类型调整系统提示词）
- 内置模板（小说大纲/剧本格式/分镜脚本）
- 管理后台（用户管理、统计概览、系统设置）

### 核心差距
平台目前是一个**「编辑器 + 聊天框」**，而 NotebookLM 是一个**「素材驱动的 AI 研究助手」**。差距不在 UI，在于**缺少素材层和基于素材的 AI 能力**。

## 三、功能规划（按优先级分层）

### P0 — 核心体验补齐（必须做）

#### 1. 素材源系统（Source System）
> NotebookLM 的灵魂功能。用户上传参考资料，AI 基于素材回答。

**功能描述：**
- 在任务级别增加「素材」面板（左侧边栏新增 Tab 或独立页面）
- 支持的素材类型：
  - 粘贴文本
  - 上传 TXT / Markdown 文件
  - 导入网页 URL（自动抓取正文）
  - 导入已有章节作为素材
- 每个任务最多 30 个素材源
- 素材列表展示：名称、类型、字数、导入时间
- 支持删除、重命名素材

**后端需要：**
- 新增 `Source` 模型（id, task_id, name, type, content, metadata, created_at）
- 素材存储在数据库中（文本形式）
- 新增素材 CRUD API

**前端需要：**
- 素材管理面板组件
- 文件上传 / URL 导入 / 文本粘贴三种导入方式

---

#### 2. 源关联 AI 对话（Source-Grounded Chat）
> 让 AI 回答基于用户上传的素材，而不是凭空发挥。

**功能描述：**
- AI 对话时自动检索当前任务的所有素材
- 在系统提示词中注入素材摘要/全文
- 回答中标注引用来源（如「根据《九州风云录大纲》第三章...」）
- 可选择本次对话关注哪些素材（可选）

**后端需要：**
- 修改 WebSocket chat 逻辑：发送前查询关联素材
- 系统提示词模板升级：加入素材上下文
- 简单的关键词匹配或全文检索（初期不需要向量数据库）

**前端需要：**
- 对话气泡中展示引用标签
- 点击引用可查看素材原文

---

#### 3. 章节版本历史（Version History）
> NotebookLM 有 source 版本管理，创作平台更需要内容版本回溯。

**功能描述：**
- 每次保存章节时，自动存一份历史版本（已有 version 字段，但未存储历史内容）
- 在编辑器工具栏增加「历史版本」按钮
- 弹窗展示版本列表（版本号、保存时间、内容预览）
- 支持「查看」某个版本（只读对比）
- 支持「恢复」某个版本

**后端需要：**
- 新增 `ChapterVersion` 模型（id, chapter_id, version, title, content, created_at）
- 保存章节时自动写入版本记录
- 新增版本列表 API 和恢复 API

---

#### 4. 自动摘要与大纲生成
> NotebookLM 上传素材后自动生成摘要，创作平台应在任务/章节维度自动生成。

**功能描述：**
- 创建任务时，可选择让 AI 自动生成故事大纲
- 在章节列表底部增加「AI 摘要」面板
- 对已有章节内容自动生成：
  - 一句话摘要
  - 关键人物列表
  - 主要事件点
- 摘要随章节保存自动更新（可手动刷新）

**后端需要：**
- 新增摘要生成 API（调用 LLM，输入章节全文）
- `Chapter` 模型增加 `summary` 字段（或独立表）
- 任务级别的整体摘要 API

---

### P1 — 体验增强（强烈建议）

#### 5. 导出功能
> 创作完成后，用户需要将作品导出为通用格式。

**功能描述：**
- 导出单章 / 全部章节
- 支持格式：
  - Markdown（.md）
  - 纯文本（.txt）
  - PDF（适合打印/分享）
  - DOCX（适合投稿）
- 导出时可选择是否包含 AI 对话记录

**技术方案：**
- 前端使用 `jsPDF` 或 `html2pdf` 生成 PDF
- 后端使用 `python-docx` 生成 DOCX
- 纯前端即可完成 MD/TXT 导出

---

#### 6. 任务内搜索
> 当章节多了之后，快速定位内容是刚需。

**功能描述：**
- 全局搜索框（顶部或 Ctrl+K 快捷键）
- 搜索范围：当前任务的所有章节标题 + 正文
- 搜索结果展示：章节名、匹配片段高亮
- 点击结果跳转到对应章节

**技术方案：**
- 前端本地搜索（章节内容已加载）
- 或后端提供搜索 API（SQLite LIKE / FTS5）

---

#### 7. 快捷键支持
> 效率工具的基本素养。

**功能描述：**
- `Ctrl+S` — 保存当前章节
- `Ctrl+N` — 新建章节
- `Ctrl+K` — 搜索
- `Ctrl+B` — 加粗（编辑器内）
- `Ctrl+Enter` — 发送对话消息
- `Esc` — 关闭弹窗/取消操作
- 在页面底部或设置中展示快捷键列表

---

#### 8. 用户个人设置
> 当前用户只能退出，无法管理自己的信息。

**功能描述：**
- 个人设置页面（或下拉菜单）
- 修改姓名
- 修改密码（需输入旧密码）
- 查看账号信息（邮箱、注册时间）
- 主题切换（深色 / 浅色）— 此项在 NotebookLM 风格重设计后为浅色主题

---

### P2 — 差异化功能（锦上添花）

#### 9. AI 一键续写 / 改写
> 不只是对话，直接在编辑器中调用 AI。

**功能描述：**
- 选中一段文字，右键出现 AI 菜单：
  - 「续写」— 从光标位置继续写
  - 「改写」— 改变风格/语气
  - 「扩写」— 扩展细节
  - 「缩写」— 精简内容
  - 「翻译」— 中英互译
- AI 生成的内容以 diff 形式预览，用户确认后插入

---

#### 10. 角色与设定管理
> 小说创作的核心需求：管理人物关系和世界观。

**功能描述：**
- 在任务中增加「设定」Tab（与章节、素材并列）
- 角色卡片：姓名、外貌、性格、背景故事、关系网络
- 世界观设定：地图、势力、规则体系
- AI 对话时自动注入角色设定作为上下文
- 可视化人物关系图（简单版：列表；进阶版：力导向图）

---

#### 11. 协作功能
> NotebookLM 支持笔记本分享，创作平台更需要多人协作。

**功能描述：**
- 任务分享：生成分享链接，他人可查看/编辑
- 权限控制：只读 / 可编辑 / 管理员
- 协作者列表展示
- 实时协同编辑（进阶，可先做异步）

---

#### 12. AI 多模态能力扩展
> 参考 NotebookLM 的音频概览，拓展 AI 输出形态。

**功能描述：**
- **故事播客**：将章节内容转为双人对话式音频概览（TTS）
- **角色配音**：为不同角色生成不同声线的朗读
- **分镜可视化**：为分镜类型任务生成简易配图（调用图像生成 API）
- **思维导图**：自动从章节内容提取关键概念生成思维导图

---

### P3 — 管理与运营（后台增强）

#### 13. 管理后台增强

**当前不足：**
- 统计只有用户数和任务数
- 无法查看用户的实际内容
- 无操作日志

**增强内容：**
- 增加统计维度：章节数、对话数、活跃用户数、每日新增
- 任务浏览：管理员可查看任意任务的内容（只读）
- 操作日志：记录关键操作（用户创建/禁用、任务删除等）
- 系统健康：LLM 调用次数、平均响应时间、错误率

---

#### 14. 模板管理增强

**当前不足：**
- 模板只读，无法自定义
- 管理后台有模板列表但无编辑功能

**增强内容：**
- 用户可创建自定义模板
- 模板市场：用户可将自己的模板分享到公共库
- 创建任务时可选择模板自动填充章节结构
- 管理员可编辑/删除内置模板

## 四、技术架构建议

### 当前架构的局限
1. **无向量检索**：素材关联 AI 目前用关键词匹配够用，但素材量大后需要向量数据库（如 ChromaDB / Qdrant）
2. **SQLite 单文件**：适合初期，多人协作时可能成为瓶颈
3. **无文件存储**：所有内容存在数据库文本字段，大文件（PDF、音频）需要对象存储

### 建议的演进路径
```
Phase 1 (P0): 素材系统 + 源关联AI + 版本历史 + 自动摘要
  └─ 纯文本素材，SQLite 够用，LLM 直接注入上下文

Phase 2 (P1): 导出 + 搜索 + 快捷键 + 用户设置
  └─ 前端增强为主，后端改动小

Phase 3 (P2): AI 一键操作 + 角色管理 + 协作
  └─ 需要 WebSocket 多端同步，考虑引入 Redis

Phase 4 (P3): 管理增强 + 模板市场 + 多模态
  └─ 可能需要对象存储（MinIO/OSS）和 TTS 服务
```

## 六、与 NotebookLM 的差异化定位

NotebookLM 是**通用研究助手**，面向学生、研究员、知识工作者。
本平台是**创意写作工具**，面向小说家、编剧、分镜师。

**我们独有的：**
- 三种创作类型的专业化支持（小说/剧本/分镜）
- 章节化的内容组织（NotebookLM 是扁平的素材列表）
- 版本历史与内容回溯
- 编辑器 + 预览的双栏写作体验
- 角色与世界观管理（NotebookLM 没有）

**从 NotebookLM 学习的：**
- 素材驱动的 AI 交互模式
- 引用溯源机制
- 自动生成多种输出（摘要/FAQ/时间线）
- 简洁明亮的 UI 设计语言

**不必照搬的：**
- 音频概览（创作平台更关注文字，可作为 P2 远期功能）
- 向量检索（初期关键词匹配足够）
- 50+ 素材源（创作场景 30 个足够）
