# Best Option

维护当前阶段对 AI 工具、框架和方法的推荐结论。

## 使用规则

- 这里只保留当前有效的推荐
- 如果推荐变化，原因写入 `knowledge/decisions/`
- 每条推荐都要带适用场景和不适用条件

## Current Focus

### Learning Strategy

- 推荐：优先吸收官方文档、发布日志和高质量工程实践
- 适用：需要维护长期有效的 AI 知识体系
- 不适用：只追短期热点新闻的场景

### Knowledge Structure

- 推荐：采用 `主题卡片 + 决策记录 + 最佳方案总表`
- 适用：需要持续更新、支持讨论和追踪结论变化
- 不适用：一次性读后即弃的信息整理

### Discussion Mode

- 推荐：对高影响变化先讨论再更新长期结论
- 适用：工具与框架变化频繁、存在选型成本的研究场景
- 不适用：仅做简单资料摘录的场景

### Agent And Bot Boundary

- 推荐：将复杂推理、归因和多步任务交给 Agent，将简单查询、固定推送和常规解释交给 Bot
- 适用：需要同时兼顾可靠性、效率和专业判断的知识与数据系统
- 不适用：所有任务复杂度都接近，且没有明显执行边界的场景

### Knowledge Access

- 推荐：优先采用 `RAG + MCP` 作为知识接入路径，避免依赖模型内隐记忆
- 适用：要求可追溯、低幻觉、可持续更新的知识型 Agent
- 不适用：完全离线、且知识范围极小的单次任务

### Retrieval Strategy

- 推荐：采用 `向量检索 + 关键词检索 + 融合排序` 的混合检索策略
- 适用：文档类型复杂、既要语义相关又要保留精确术语命中的场景
- 不适用：数据规模很小、关键词即可覆盖的场景
