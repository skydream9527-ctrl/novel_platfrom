# Data Intelligence System

## 来源

- 飞书文档：`基于大模型的数据智能体系规划`
- 获取时间：2026-04-21

## 核心结论

1. 数据智能体系应拆分为三层：`Agent-Skill`、`NL-SQL/中间表`、`RAG 知识库`。
2. Agent 负责复杂任务，包括归因、策略理解、多步分析和专业经验封装。
3. 机器人更适合承担简单查询、日报周报、基础解释等稳定高频任务。
4. 知识库不只是资料仓库，而是 Agent 和 Bot 的事实基础，需持续更新与评估。
5. MCP 适合作为知识库和 Agent 的标准化连接层，让检索、查询、更新能力可调用。

## 可复用设计

### Agent 边界

- 复杂归因分析交给 Agent
- 简单数据查询交给 Bot
- 不把所有任务都塞进同一个执行链

### RAG 设计

- 按语义单元切分文档
- Chunk 控制在 `500-1000 tokens`
- 保留适度重叠，避免语义断裂
- 使用中文优化 embedding
- 采用混合检索而不是只靠向量召回

### 更新机制

- 自动更新：结构化知识
- 半自动更新：案例、FAQ、复盘
- 定期巡检：识别过期内容并刷新

## 对本 Agent 的启发

1. 本 Agent 也应区分学习、知识沉淀、主动巡检三类职责。
2. 最佳方案不只写推荐，还要写边界和切换条件。
3. 知识维护要支持版本化，而不是堆积静态摘要。

## 待讨论

1. 后续是否要把本 Agent 也拆成 `research bot + decision agent` 两层。
2. 是否引入更强的结构化索引，而不是只靠 markdown 文件。
