# Tooling And Framework Selection

## 来源

- 飞书文档：第二篇已学习文档
- 获取时间：2026-04-21

## 当前理解

1. 工具和框架学习不能停留在“知道名字”，必须落到使用边界、组合方式和替代关系。
2. 在 Agent 系统中，框架不是目的，可靠工作流、可维护结构和可更新知识才是目的。
3. 更小、更稳定的工作流通常优于过度复杂的多 Agent 设计。

## 选型原则

- 优先选择文档清晰、边界明确、可调试的框架
- 优先保留最少必要抽象，避免过早引入复杂编排层
- 把工具调用成功率、错误恢复和结果校验作为首要指标
- 对新框架保持跟踪，但不要轻易替换当前稳定方案

## 对本 Agent 的启发

1. 本仓库维护的“最佳方案”应以稳定和可复用为主，而不是追逐最新名词。
2. 需要长期记录：推荐主方案、候补方案、何时切换。
3. 工具与框架的知识卡片应始终包含“不适用条件”。
