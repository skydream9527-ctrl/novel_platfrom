ICE Data Workbench 使用指南

欢迎使用 ICE Data Workbench — AI 驱动的数据工作流工作台。本指南帮助你快速上手，了解全部核心功能。

#一、快速开始

从 Dashboard 开始你的第一个任务。

1.1 创建第一个任务

在 Dashboard 的 ⚡ 快速开始区，选择一个适合你场景的工作范式：

  • ⚖ AB 实验 — 实验设计、分流、显著性检验、报告
  • 📈 经营分析 — 经营数据拆解归因
  • 🔌 版本灰度 — 灰度放量监控
  • 📊 数据分析 — 自由查询与探索
  • 🔥 波动分析 — 指标异动归因

或者点击 + 开放任务 创建不绑定范式的自由任务。

1.2 选择工作范式

每个范式都预绑定了一个专属 Agent 和一组 Skills。范式 = 任务模板 + 智能体能力的组合。

#二、Agent 系统

Agent 是平台的核心智能体。每个 Agent 通过对话持续进化，越用越聪明。

2.1 5 个内置 Agent

Agent范式主要能力
实验分析 AgentAB 实验分流 / 指标 / 显著性 / 报告
经营洞察 Agent经营分析指标拆解 / 趋势 / 归因 / 报告
灰度监控 Agent版本灰度灰度查询 / 版本对比 / 质量评估
数据分析 Agent数据分析SQL / 清洗 / 可视化 / 建模
波动归因 Agent波动分析异动检测 / 多维下钻 / 归因

2.2 经验卡片机制

对话中的高质量结论会被自动提炼为 经验卡片，经管理员审批后注入 Agent 的 System Prompt：

flow用户对话 → 高质量结论 → AgentMemory
       → 多次验证 → 经验卡片(draft)
       → admin 审批 → approved
       → 注入 System Prompt
       → Agent 后续行为进化

#三、对话与工具

在工作空间中你可以：

  • 📄 引用文件 / 知识库 — 让 Agent 读取上下文
  • ⚡ 引用技能 — 强制 Agent 使用某个 Skill
  • 📎 上传新文件 — 拖拽或点击上传，最大 20MB

3.3 Tool Calling

Agent 调用 Skill 时会显示 工具调用卡片，包含：

  • ⏳ 执行中状态
  • ✅ 完成 — 显示返回行数 / 摘要 / 文件 / 图表
  • ❌ 失败 — 显示错误码 + [🔁 重试] [📋 复制错误] [🚩 反馈]
  • ⏱ 超时（30s）

#四、知识库

支持 飞书知识库和 Mify RAG，每日自动同步。在工作空间侧栏点击 🔄 可手动同步。

#五、协作与共享

把任务共享到公共区，其他人即可加入协作（默认 collaborate 权限）。

#六、定时任务

支持 cron 表达式自动执行：每日 09:00、每周一、每月 1 日等。
