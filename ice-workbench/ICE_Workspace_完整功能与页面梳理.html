⑪Tool Calling 完整执行流程

用户消息 (含 referenced_file_ids / referenced_skill_ids)
   ↓
组装 LLM 请求
  · System Prompt = Agent.config.system_prompt
                   + "## 经验规则" + 已批准经验卡片 (最多 20 条)
                   + "## 可用工具" + Skill 描述
  · tools[] = agent_skills 绑定 + 用户引用 Skill 的 tool_schema
  · 文件内容注入: CSV ≤100 行全量；>100 行 schema+前 20 行；MD/TXT 全量
   ↓
httpx SSE 调用 LLM
   ↓
├── 文本 chunk → ws agent_message
└── tool_calls → 执行循环 (≤ 5 轮):
      ↓ ws tool_call_start
      ToolExecutor:
        1. 查 tool_entry → 动态 import
        2. SQL 类先过安全审计 (allow / warn / block)
        3. 执行函数 (共享 ToolExecutionContext)
           shared_data[kyuubi_sql_query / auto_drilldown / ...]
        4. 单 Tool 30s 超时
      ↓ ws tool_call_done (含 chart_config?)
      result 回传 LLM → 继续生成
      ├── 又 tool_calls → 继续 (≤5 轮)
      └── 文本 → ws agent_message → done

ToolExecutionContext 跨 Skill 共享上下文，跨对话隔离。

⑫Agent 进化机制 · 经验卡片

用户对话 → Agent 回复
   ↓
异步分析对话质量:
  · 用户确认有效的分析路径
  · 高质量回答 (👍 标记)
  · 反复被引用的结论
   ↓
提炼写入 AgentMemory
   ↓
记忆积累 / 多次验证 → 自动升级为 ExperienceCard(draft)
   ↓
管理员审批 → approved
   ↓
注入 Agent System Prompt 的 "经验规则" 章节 (最多 20 条)
   ↓
Agent 后续对话行为进化
