## 1. 分析框架

### 1.1 MECE 分析法

- Mutually Exclusive, Collectively Exhaustive (互斥且穷尽)
- 拆解问题时确保子项不重叠、不遗漏
- 适用场景: 指标拆解、原因归因、方案列举

### 1.2 漏斗分析

- 将用户行为路径抽象为多步漏斗
- 关注每步转化率和流失原因
- 适用场景: 注册转化、付费转化、功能使用漏斗

### 1.3 对比分析

| 对比维度 | 说明 |
|---------|------|
| 时间对比 | 环比、同比、趋势变化 |
| 群组对比 | 不同用户群体的行为差异 |
| 版本对比 | A/B 实验、版本迭代效果 |
| 竞品对比 | 行业基准对标 |

### 1.4 归因分析

- 多维下钻: 按维度逐层拆解，定位变化根因
- 贡献度分析: 各维度对总变化的贡献占比
- 相关性分析: 探索指标间关联关系

## 2. 常用分析方法

### 2.1 描述性分析

- 数据概览: 均值、中位数、分位数、标准差
- 分布分析: 直方图、箱线图、正态性检验
- 时序趋势: 移动平均、季节性分解

### 2.2 诊断性分析

- 异常检测: 3σ 原则、IQR 方法、孤立森林
- 根因分析: 决策树、多维下钻、关联规则
- 影响因素: 相关分析、方差分析

### 2.3 预测性分析

- 趋势预测: 线性回归、ARIMA、Prophet
- 分类预测: 逻辑回归、随机森林
- 聚类分析: K-Means、DBSCAN

## 3. 指标体系设计

### 3.1 北极星指标

- 反映产品核心价值的单一指标
- 特征: 可衡量、可指导行动、反映用户价值
- 示例: DAU、GMV、Weekly Active Subscribers

### 3.2 指标分层

```
L0: 北极星指标 (1 个)
    ↓
L1: 一级业务指标 (3-5 个)
    ↓
L2: 二级过程指标 (每个 L1 拆解为 3-5 个)
    ↓
L3: 操作指标 (可直接影响的可操作指标)
```

### 3.3 指标定义规范

| 字段 | 说明 |
|------|------|
| 指标名称 | 清晰简洁 |
| 业务含义 | 反映什么业务现象 |
| 计算公式 | 精确的数学定义 |
| 数据来源 | 哪张表、哪个字段 |
| 统计口径 | 时间窗口、用户范围、去重规则 |
| 更新频率 | 实时/小时/天/周 |

## 4. 数据质量

### 4.1 数据验证清单

- [ ] 数据完整性: 是否有缺失值、NULL 比例
- [ ] 数据一致性: 同一指标不同来源是否一致
- [ ] 数据时效性: 数据是否按时更新
- [ ] 数据准确性: 抽样验证计算逻辑

### 4.2 常见数据陷阱

| 陷阱 | 说明 | 应对 |
|------|------|------|
| 幸存者偏差 | 只看到存活样本 | 确认分析群体是否完整 |
| 辛普森悖论 | 分组结论与总体相反 | 检查是否需要控制混淆变量 |
| 采样偏差 | 样本不代表总体 | 验证采样方法的代表性 |
| 指标膨胀 | 口径不一致导致虚高 | 统一指标定义和统计口径 |

## 5. 分析报告规范

### 5.1 报告结构

```
1. 背景与目标 (为什么分析)
2. 数据说明 (数据来源、时间范围、样本量)
3. 分析方法 (使用了什么方法)
4. 核心发现 (3-5 个关键发现，数据支撑)
5. 结论与建议 (明确的行动建议)
6. 附录 (详细数据表格、补充分析)
```

### 5.2 可视化规范

- 折线图: 趋势变化
- 柱状图: 分类对比
- 饼图: 占比分布（类别 ≤ 6 个）
- 散点图: 相关关系
- 热力图: 多维度交叉分析
